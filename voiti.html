Кратко: что такое HTML

HTML (HyperText Markup Language) — язык разметки для создания структуры веб-страниц. Он задаёт семантические блоки (заголовки, абзацы, списки, таблицы, формы и т.д.), которые браузер отображает, а поисковые системы и вспомогательные технологии (например, скрин-ридеры) используют для понимания содержимого.

Основные принципы

  1. Семантика — используйте теги по назначению: <nav> для навигации, <footer> — для футера, <section> для разделов.
  2. Отделение структуры от представления — HTML для структуры, CSS для визуала, JS для поведения.
  3. Доступность — добавляйте alt, role, aria-атрибуты и правильно ставьте заголовки.

Быстрый справочник по тегам

<div><span><a><img> <header><main><article><section> <nav><footer><form><input> <button><ul><li><table><svg>

Пример семантической обвязки

<!doctype html>
<html lang="ru">
  <head>…</head>
  <body>
    <header><nav>…</nav></header>
    <main>
      <article>
        <h1>Заголовок</h1>
        <p>Контент</p>
      </article>
    </main>
    <footer></footer>
  </body>
</html>

Частые атрибуты

  • id, class — для стилизации/скриптов
  • alt — альтернативный текст для изображений
  • aria-* — для доступности
  • role — определяет роль элемента

Формы

<form action="/" method="post">
  <label>Email<input type="email" name="email" required></label>
  <button type="submit">Отправить</button>
</form>

Полезные практики и советы

  • Всегда указывайте <!doctype> — иначе режим отображения может сломаться.
  • Используйте валидатор — проверяйте HTML на validator.w3.org.
  • Доступность — чёткая структура заголовков, контраст текста, клавиатурная навигация.
  • SEO — мета-теги, семантическая разметка, правильные заголовки.
  • Производительность — оптимизируйте изображения, ленивая загрузка (loading="lazy"), минимизируйте DOM.
  • Безопасность — внимательно с target="_blank" rel="noopener noreferrer".

Встроенные возможности браузера

Современные браузеры дают HTML API: picture для отклика изображений, video/audio для мультимедиа, dialog для модальных окон, геолокация, localStorage, service workers (вместе с HTTPS) и многое другое.

Пример — прогрессивное изображение

<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="Описание" loading="lazy">
</picture>

Доступность (Accessibility)

Доступность — один из важнейших критериев. Коротко:

  • Используйте семантические теги.
  • Добавляйте alt к изображениям, label к элементам форм.
  • Проверяйте фокусную навигацию (только клавиатура).
  • ARIA — полезно, но не заменяет семантику.

SEO & Best practices

  • Уникальные иеррархические заголовки (H1 → H2 → H3).
  • Метаданные: <meta name="description" content="...">, Open Graph, Twitter cards.
  • Мобильная оптимизация и скорость — критично для ранжирования.

Чек-лист перед релизом

  1. Пройти HTML-валидатор
  2. Проверить на мобильных устройствах
  3. Проверить доступность (axe, lighthouse)
  4. Оптимизировать изображения и шрифты
  5. Настроить мета-теги и sitemap
